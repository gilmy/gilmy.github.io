接口

说到 Java 就不得不提到 封装、继承、多态 三大特性; 而谈到 多态 又绕不开 接口 和 抽象类 这个话题. 甚至在面试中这两位都是常客, 经常会有面试官问你 接口 和 抽象类 的区别. 这里可以先自己想一下区别有哪些 ?

Java 8 之前

暂时不管为什么标题要写 Java 8, 如果你在搜索引擎搜索 接口和抽象类的区别是什么 大概率的回答是如下表述的:

  1. 接口中的方法默认是 public , 所有的方法在接口中不能有实现, 而抽象类可以有非抽象的方法;
  2. 接口中除了 static 、 final 变量, 不能有其他变量, 而抽象类中则不一定;
  3. ⼀个类可以实现多个接⼝, 但只能实现⼀个抽象类; 接⼝⾃⼰本身可以通过 extends 关键字扩展多个接⼝;
  4. 接⼝⽅法默认修饰符是 public, 抽象方法可以有 public 、 protected 这些修饰符;
  5. 从设计层面来说, 抽象类是对类的抽象, 是一种模板设计; 而接口是对行为的抽象, 是一种行为的规范.

如上回答在 Java 8 之前版本中没有任何问题, 但如果是对于 Java 8 及之后的版本来说的话, 就不太正确或者说是不太完善了…

Java 8

在 Java 8 之前接口只能有两种东西, 即 常量变量和抽象方法 , 在 Java 8 版本中引入了一些新特性, 即 默认方法和静态方法

为什么要引入默认方法和静态方法

知其然, 还得知其所以然.

  1. 先说说为什么要引入默认方法:
    我们知道接口的作用主要是定义该类型实例具有的功能, 也是为了支持运行时动态方法的解析. 所有在接口中定义的方法, 都必须由实现类来完成具体的实现(Java 8 之前).
    问题来了, 假设我们要在一个老旧的系统之上修改一个接口, 拓展一个新的方法该怎么办 ?
    在Java 8 以前, 如果接口新增了一个方法, 那么我们必须在所有的实现类中都去添加该方法的实现, 否则编译是无法通过的. 如果实现类比较多, 或者说实现类不可控(不在我们手上), 那就很麻烦了…
    而 默认方法 的出现就是为了解决这个问题, 它提供了一个实现, 当没有显式地提供其他实现时就采用默认的这个实现, 这样添加新的方法就不会破坏现有的实现代码, 只需要更改我们需要更改的实现类重写这个新增方法即可.
  2. 再说说静态方法:
    这个引入的初衷就简单的多了, 可以帮助我们避免或者省去实现类中不太理想的实现, 例如在Map接口中有几个静态的comparingXXX接口.

Java 9

在 Java 9 中接口中又引入了 私有方法和私有静态方法
这个就好理解多了, 即然允许了具体的方法实现, 那么久而久之必然会产生重复代码, 这时候我们就需要提高代码的重用、减少重复代码, 那么私有方法的重现也就是必然的了.

总结

回顾上面的面试题, 我们该这样来回答

  1. 接口中的方法默认是 public 还有 default 、 static 、 private 修饰符, 默认方法和静态方法在接口中实现, 默认方法可以被实现类重写, 静态方法不能被重写; 而抽象类可以有非抽象的方法;
  2. 接口中除了 static 、 final 变量, 不能有其他变量, 而抽象类中则不一定;
  3. ⼀个类可以实现多个接⼝, 但只能实现⼀个抽象类; 接⼝⾃⼰本身可以通过 extends 关键字扩展多个接⼝;
  4. 接⼝⽅法默认修饰符是 public, 抽象方法可以有 public 、 protected 这些修饰符;
  5. 从设计层面来说, 抽象类是对类的抽象, 是一种模板设计; 而接口是对行为的抽象, 是一种行为的规范.
